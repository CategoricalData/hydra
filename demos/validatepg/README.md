# ValidatePG demo - property graph validation

A translingual demo that validates property graphs against a schema, running the same
Hydra validation logic in Haskell, Java, and Python.

## Quick start

```bash
cd demos/validatepg
./bin/run.sh
```

This will:

1. Build the Java project
2. Generate example data (a schema and twelve graphs) using the Java PG DSL
3. Run validation in each host language
4. Compare outputs across hosts and display a timing summary

## What it demonstrates

- A property graph schema defined using the Java DSL (`hydra.pg.dsl`)
- Graph instances — one valid and eleven invalid — encoded to JSON via `hydra.encode.pg.model`
- Validation via `hydra.validate.pg.validateGraph` in all three host languages
- Identical results across Haskell, Java, and Python

## Validation conditions covered

| Graph file | Condition |
|------------|-----------|
| `valid_social_network` | All checks pass |
| `missing_required_property` | Vertex missing a required property |
| `wrong_id_type` | Vertex id type does not match schema |
| `unknown_edge_endpoint` | Edge references a non-existent vertex |
| `unexpected_vertex_label` | Vertex label not defined in schema |
| `unexpected_edge_label` | Edge label not defined in schema |
| `property_value_type_mismatch` | Property value type does not match schema |
| `unexpected_property_key` | Property key not defined in schema |
| `wrong_in_vertex_label` | Edge in-vertex has wrong label |
| `wrong_out_vertex_label` | Edge out-vertex has wrong label |
| `missing_required_edge_property` | Edge missing a required property |

## Using your own data

Supply your own schema and graph JSON files:

```bash
./bin/run.sh --data-dir /path/to/your/data
```

The data directory should contain:

- `schema.json` — a `GraphSchema<LiteralType>` encoded via `hydra.encode.pg.model`
- One or more `<name>.json` files — `Graph<Literal>` instances encoded the same way

See the generated files in `/tmp/hydra-validatepg-*` for the expected JSON format.

## Directory structure

```
demos/
  validatepg/
    README.md                                   # This file
    bin/run.sh                                  # Orchestrator script
  src/main/
    haskell/Hydra/Demos/ValidatePg/
      Demo.hs                                   # Haskell driver
    java/hydra/demos/validatepg/
      GenerateData.java                         # Data generator (Java DSL -> JSON)
      ValidateDemo.java                         # Java driver
      JsonPgDecoder.java                        # JSON -> PG model decoder
```

The orchestrator also supports a Python driver at
`demos/src/main/python/hydra/demos/validatepg/demo.py`; it is not
checked in today, so the `python` host path is skipped at runtime
(see `run.sh`'s "driver not found" branch).

## Data generation

The example data is generated by `hydra.demos.validatepg.GenerateData`, a Java program
that uses the Java PG DSL to define a social network schema (Person and Organization vertices,
worksAt and founded edges) along with the graphs listed above.

To generate data separately:

```bash
./gradlew :hydra-java:jar
java -cp "packages/hydra-java/build/libs/*.jar" \
  hydra.demos.validatepg.GenerateData /path/to/output
```

## Prerequisites

- Java 11+
- Stack (for the Haskell driver)
- Python 3.12+
- Generated ext demo modules (run `bin/sync-python.sh` if needed)
